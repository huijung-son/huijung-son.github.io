### 1. C 자료형

| 자료형         | 크기      | 값의 표현범위                                                 |
|-------------|---------|---------------------------------------------------------|
| char        | 1바이트    | -128 ~ +127                                             |
| short       | 2바이트    | -32,768 ~ +32,767                                       |
| int         | 4바이트    | -2,147,483,648 ~ +2,147,483,647                         |
| long        | 4바이트    | -2,147,483,648 ~ +2,147,483,647                         |
| long long   | 8바이트    | -9,223,372,036,854,775,808 ~ +9,223,372,036,854,775,807 |
| float       | 4바이트    | ±3.4x10^-37 ~ ±3.4x10^+38                               |
| double      | 8바이트    | ±1.7x10^-307 ~ ±1.7x10^+308                             |
| long double | 8바이트 이상 | double 이상의 표현범위                                         |

| 데이터 타입             | 리터럴 |
|--------------------|-----|
| unsigned int       | U   |
| long               | L   |
| unsigned long      | UL  |
| long long          | LL  |
| unsigned long long | ULL |
| float              | f   |
| double             | 없음  |
| long double        | Lf  |

### 2. C 포맷

- printf와 같은 함수는 인자의 개수와 타입을 고정된 방식으로 정의하지 않고,<br>
첫 번째 인자인 포맷 문자열에 따라 동적으로 인자를 처리합니다.<br>
C 컴파일러는 이 가변 인자 목록에 대해 타입 검사를 수행하지 않으므로, 잘못된 포맷 형식을 지정해도 컴파일 에러가 발생하지 않는다.
[GCC](https://gcc.gnu.org/onlinedocs/gcc/Warning-Options.html#index-Wformat)

| 형식   | 설명                           |
|------|------------------------------|
| %d   | 부호 있는 10진 정수 출력              |
| %c   | 단일 문자 출력                     |
| %p   | 포인터(메모리 주소) 출력               |
| %x   | 소문자 형태의 16진수 정수 출력           |
| %X   | 대문자 형태의 16진수 정수 출력           |
| %o   | 8진수 정수 출력                    |
| %s   | 문자열 출력                       |
| %u   | 부호 없는 10진 정수 출력              |
| %ld  | long 타입 부호 있는 10진 정수 출력      |
| %lld | long long 타입 부호 있는 10진 정수 출력 |
